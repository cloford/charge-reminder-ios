# ChargeReminder

ChargeReminder は、就寝前のiPhone充電忘れを防ぐための iOS アプリMVPです。

最初の案は次のようなものでした。

> 0時を超えた時に、iPhoneのバッテリーが50%以下なら通知する。

ただし、iOSのバックグラウンド実行制約を考慮すると、アプリが0時ちょうどに起動してバッテリー残量を判定し、条件付きで通知を出すことは現実的ではありません。

そのため、MVPでは次の仕様に整理しています。

> 指定した時刻に充電確認通知を出し、アプリ起動時・復帰時にバッテリー状態を確認する「充電忘れ防止リマインダー」。

## 現在の状態

このリポジトリには、初期実装の土台が入っています。

- SwiftUIアプリの骨格
- 4タブ構成
- ローカル通知の許可取得と通知予約
- 最大3件の通知時刻設定
- バッテリー残量・充電状態表示
- UserDefaultsによる設定保存
- 簡易的な充電推奨判定
- 充電習慣スコアの基本モデル

この作業はWindows上で準備したため、Xcodeでのビルド確認とiPhone実機検証はまだ行っていません。

次の作業にはMacとXcodeが必要です。

## アプリの目的

就寝前や深夜に、iPhoneの充電忘れを防ぐことが目的です。

想定ユーザー:

- 寝落ちしやすい人
- 寝る前にiPhoneの充電を忘れがちな人
- 翌朝のバッテリー不足で困った経験がある人
- 充電習慣を作りたい人

このアプリは目覚ましアラームの代替ではありません。

## MVPの範囲

MVPとは、ユーザーに価値を届けられる最小限の製品です。

このアプリでは、まず「深夜の充電忘れ防止」が成立することを最優先にしています。

### MVPに入れる機能

- 最大3件の通知時刻を設定できる
- 初期通知時刻:
  - 22:00
  - 23:00
  - 0:00
- 各通知のON/OFF
- 各通知時刻の変更
- 起床予定時刻の設定
- バッテリーしきい値の設定
  - 初期値: 50%
- 指定時刻にローカル通知を出す
- アプリ起動時・復帰時にバッテリー残量と充電状態を表示する
- 状態に応じて「安全」「注意」「充電推奨」を表示する
- 充電習慣スコアを表示する
- iOS制約をアプリ内で説明する

### MVPでやらないこと

- iPhone標準の時計アプリのアラーム操作
- 必ず鳴るアラームの保証
- バックグラウンドでの常時バッテリー監視
- 0時ちょうどにバッテリー50%以下かを判定して通知を出し分ける挙動
- 高精度なバッテリー消費予測
- 自由な条件ルール作成
- iCloud同期
- Apple Watch対応

## iOS上の重要な制約

このアプリは、以下のiOS制約を前提に設計しています。

- 通常のiOSアプリは、バックグラウンドで常時動き続けることはできません。
- アプリが0時ちょうどに起きて、現在のバッテリー残量を読める保証はありません。
- ローカル通知は指定時刻に予約できますが、通知許可・集中モード・iOS通知設定の影響を受けます。
- バッテリー残量や充電状態は、アプリが起動中または復帰した時に取得します。
- 標準の時計アプリのアラームを、このアプリから自由に作成・変更する仕様にはしません。

そのため、MVPの挙動は次のようになります。

1. 指定した時刻に充電確認通知を出す。
2. ユーザーがアプリを開いた時、またはアプリに戻った時にバッテリー状態を確認する。
3. ホーム画面に現在の状態と充電推奨を表示する。

## 技術方針

| 項目 | 内容 |
|---|---|
| 対象 | iOS |
| UI | SwiftUI |
| 設計 | 軽量MVVM |
| 保存 | UserDefaults |
| 通知 | UserNotificationsによるローカル通知 |
| バッテリー取得 | UIDevice |
| Background Tasks | MVPでは使わない |
| Bundle ID | `com.cloford.chargereminder` |

## 画面構成

MVPでは4タブ構成にしています。

| タブ | 役割 |
|---|---|
| ホーム | バッテリー残量、充電状態、充電推奨、次回通知、今日のスコアを表示 |
| 通知 | 3件の通知時刻、ON/OFF、時刻変更を管理 |
| スコア | 今日のスコア、直近7日平均、スコア内訳を表示 |
| 設定 | 起床時刻、しきい値、通知許可状態、iOS制約説明を表示 |

## 充電状態の表記

ユーザー向けの表示は以下に統一します。

| 内部状態 | 表示 |
|---|---|
| 未接続 | 未接続 |
| 充電中 | 充電中 |
| full | 充電完了 |
| 不明 | 不明 |

「満充電」ではなく「充電完了」と表記します。

## 充電推奨判定

初期MVPでは、以下のルールで判定します。

| 条件 | 表示 |
|---|---|
| 充電中 | 安全 |
| 充電完了 | 安全 |
| 未接続かつ80%以上 | 安全 |
| 未接続かつ50〜79% | 注意 |
| 未接続かつ49%以下 | 充電推奨 |
| 起床まで8時間以上かつ60%未満 | 充電推奨 |
| バッテリー状態が取得できない | 状態不明 |

## 充電習慣スコア

1日あたり最大3点で、充電習慣を簡易的に記録します。

| 条件 | 点数 |
|---|---:|
| 通知後にアプリを開いた | +1 |
| アプリ確認時に充電中または充電完了だった | +1 |
| 翌朝、バッテリーがしきい値以上だった | +1 |

同じ条件は1日1回だけ加点します。

翌朝判定は、起床予定時刻以降にアプリを開いた時に行います。

## App Store向けの注意

このアプリは「目覚ましアラーム」ではなく「充電忘れ防止リマインダー」として説明します。

使う表現:

- 就寝前の充電忘れを防ぐリマインダー
- 指定した時刻に充電確認を促します
- アプリ起動時・復帰時にバッテリー状態を確認できます
- 通知はiOSの通知設定に依存します
- 目覚ましアラームの代替ではありません

避ける表現:

- 必ず通知します
- 必ず起こします
- 目覚ましアラームの代わりになります
- 0時にバッテリー50%以下なら自動通知します
- バックグラウンドで常時監視します
- バッテリー切れを完全に防止します

## Mac購入後にやること

次回、MacとXcode環境を用意した後に行う作業です。

1. GitHubからこのリポジトリをcloneする。
2. Xcodeで `ChargeReminder.xcodeproj` を開く。
3. Apple Developer Teamを設定する。
4. ビルドする。
5. ビルドエラーがあれば修正する。
6. iPhone SE 第2世代に実機インストールする。
7. 以下を確認する。
   - 通知許可ダイアログ
   - 指定時刻のローカル通知
   - 通知ON/OFF
   - バッテリー残量表示
   - 充電状態表示
   - アプリ復帰時の状態更新

テスト、GitHub Actions、TestFlight配布の具体的な手順は `docs/testing.md` にまとめています。
