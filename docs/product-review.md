# プロダクト判断メモ

## 現時点の目的

このアプリの主目的は、ユーザーが充電忘れをしないように支援することです。

そのため、アプリは「ユーザーを採点する」よりも、「最後にいつ、どんな状態を確認したか」を分かりやすく残す方が目的に合っています。

## スコアを履歴へ変更した理由

スコア機能には以下の懸念がありました。

- 点数稼ぎができる。
- 昼に寝る人、夜勤、交代勤務などに合わない。
- 「充電忘れを防ぐ」より「点数を取る」体験に寄る。
- 加点条件の説明とQAが増える。
- MVPの主目的がぼやける。

履歴機能にすると、ユーザーは以下を確認できます。

- 最後に確認した時刻
- その時の残量
- その時の充電状態
- 通知から確認したのか、手動で確認したのか

これは、充電忘れ防止という目的により直接つながります。

## 今回のプロダクト判断

| 論点 | 判断 |
|---|---|
| スコア | MVPから外す |
| 履歴 | MVPに入れる |
| 時間帯固定 | しない |
| 昼寝/夜勤ユーザー | 排除しない |
| 通知文言 | 「寝る前」「朝」固定を避ける |
| 背景色 | 今はiOS標準に寄せる |
| 次の優先度 | 通知タップクラッシュと履歴記録の実機確認 |

## 次に検討するべきこと

1. 「起床予定時刻」という名前を維持するか。
   - 昼に寝る人を考えるなら「次の予定時刻」または「目標時刻」の方が自然かもしれません。

2. 通知プリセットを22:00/23:00/0:00のままにするか。
   - 初期値としては分かりやすいですが、昼寝/夜勤ユーザーには合いません。
   - 初回設定でユーザーに時刻を選ばせる方が自然かもしれません。

3. ホーム画面の最終確認をどこまで目立たせるか。
   - 充電忘れ防止の中心は「今の状態」なので、履歴は補助情報として扱うのがよいです。

4. 通知から起動した時にホームへ必ず誘導するか。
   - 通知の目的は状態確認なので、履歴タブではなくホームに着地する方が自然です。

5. ショートカット連携を案内するか。
   - 指定時刻に充電中かどうかをより条件付きで確認したい場合、iOSショートカット案内は有力です。
