充電忘れ防止リマインダー プロジェクト資料 v0.1

作成日: 2026-05-19 / GitHub: https://github.com/cloford/charge-reminder-ios.git / Bundle ID: com.cloford.chargereminder

1. プロジェクト概要

本プロジェクトは、就寝前や深夜にiPhoneの充電忘れを防ぐためのiOSアプリをApp Store公開することを目的とする。

初期案は「0時を超えた時に電池が50%以下なら通知」だったが、iOSのバックグラウンド制約により、MVPでは「指定時刻に充電確認通知を出し、アプリ起動時・復帰時にバッテリー状態を確認する」仕様へ整理した。

2. 課題定義

項目内容
対象ユーザー寝落ちしやすい人、就寝前に充電を忘れがちな人、翌朝のバッテリー不足で困った経験がある人。
困りごと朝にiPhoneのバッテリーが不足し、アラーム・外出・通勤・仕事に支障が出る。
解決方針寝る前から深夜にかけて段階的に充電確認を促し、アプリ上で現在の状態と充電推奨を表示する。

3. MVP定義

MVPは「ユーザーに価値を届けられる最小限の製品」とする。単に機能が少ないだけではなく、充電忘れ防止という課題を最低限解ける必要がある。

MVPに入れる機能

  • 最大3件の通知時刻設定
  • 初期通知時刻: 22:00 / 23:00 / 0:00
  • 各通知のON/OFF
  • 各通知時刻の変更
  • 起床予定時刻の設定
  • バッテリーしきい値の設定。初期値は50%
  • ローカル通知による充電確認
  • アプリ起動時・復帰時のバッテリー残量と充電状態表示
  • 安全 / 注意 / 充電推奨の判定表示
  • 充電習慣スコア
  • iOS制約の説明

MVPでやらないこと

  • 標準時計アプリのアラーム操作
  • 必ず鳴るアラーム保証
  • バックグラウンド常時バッテリー監視
  • 0時ちょうどに50%以下かを判定して通知を出し分ける挙動
  • 高精度なバッテリー消費予測
  • 自由な条件ルール作成

4. iOS制約と技術方針

論点判断
バックグラウンド実行指定時刻ぴったりにアプリが起きてバッテリー判定する保証はない。MVPではBackground Tasksを使わない。
ローカル通知UNUserNotificationCenter と UNCalendarNotificationTrigger で毎日指定時刻に通知する。
バッテリー取得UIDevice.current.isBatteryMonitoringEnabled = true を使い、起動時・復帰時に取得する。
標準アラームサードパーティアプリから時計アプリのアラームを自由に作成・変更する前提にはしない。
保存方式MVPでは UserDefaults を使う。

5. 画面仕様

画面構成は4タブで固定する。

タブ役割
ホーム現在のバッテリー状態、充電推奨、次回通知、起床時刻、今日のスコアを表示。
通知3件の通知時刻、ON/OFF、時刻変更を管理。
スコア今日のスコア、直近7日平均、スコア内訳を表示。
設定起床時刻、しきい値、通知許可状態、iOS制約説明を表示。

6. 通知仕様

  • 初期通知時刻は22:00、23:00、0:00。
  • 通知は最大3件固定。自由追加はMVP対象外。
  • 各通知はON/OFF可能。
  • 各通知時刻は変更可能。
  • 通知はバッテリー条件で出し分けず、指定時刻に充電確認として出す。
通知はiOSの通知設定、集中モード、消音設定などの影響を受ける。アプリ内・App Store説明では「必ず鳴る」と表現しない。

7. バッテリー判定仕様

条件表示
充電中安全
充電完了安全
未接続かつ80%以上安全
未接続かつ50〜79%注意
未接続かつ49%以下充電推奨
起床まで8時間以上かつ60%未満充電推奨
状態取得不可状態不明

ユーザー向け表記は「満充電」ではなく「充電完了」に統一する。

8. 充電習慣スコア

要素点数
通知後にアプリを開いた+1
アプリ確認時に充電中または充電完了だった+1
翌朝、バッテリーがしきい値以上だった+1

同じ条件は1日1回だけ加点する。翌朝判定は、起床予定時刻以降にアプリを開いた時に行う。

9. QA観点

優先度確認項目
高通知許可、指定時刻通知、通知ON/OFF、通知時刻変更。
高バッテリー残量取得、充電状態取得、充電完了表記。
高安全 / 注意 / 充電推奨の判定。
高通知未許可時の表示。
高App Store文言に過剰な保証がないこと。
中スコア、7日平均、日またぎ、端末再起動後の設定復元。

10. App Store審査リスク

避ける表現

  • 必ず通知します
  • 必ず起こします
  • 目覚ましアラームの代わりになります
  • 0時にバッテリー50%以下なら自動通知
  • バックグラウンドで常時監視します
  • バッテリー切れを完全に防止します

使う表現

  • 就寝前の充電忘れを防ぐリマインダー
  • 指定した時刻に充電確認を促します
  • アプリ起動時・復帰時にバッテリー状態を確認できます
  • 通知はiOSの通知設定に依存します
  • 目覚ましアラームの代替ではありません

11. 技術設計

領域方針
アーキテクチャ軽量MVVM。View、ViewModel、Model、Serviceに分ける。
通知NotificationService が許可取得、状態取得、予約、削除、更新を担当。
バッテリーBatteryService が残量と充電状態を取得。
設定保存SettingsStore が通知設定、起床時刻、しきい値を保存。
スコアScoreStore が日別スコアと7日平均を管理。
判定BatteryJudgement が安全 / 注意 / 充電推奨を返す。

13. 次回Mac購入後に行うこと

  1. GitHubからリポジトリをcloneする。
  2. Xcodeで ChargeReminder.xcodeproj を開く。
  3. ビルドエラーを確認・修正する。
  4. iPhone SE 第2世代に実機インストールする。
  5. 通知許可、指定時刻通知、バッテリー残量、充電状態を検証する。
  6. 検証結果をもとにスプリント2へ進む。
