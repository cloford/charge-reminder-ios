# 概要

通知タップ時のクラッシュ対策と、充電忘れ防止という目的に合わせたUX整理を行います。

主な変更:

- 通知タップ時のイベント処理を安全化
- ホーム画面の更新UXを改善
- バッテリー残量を「約xx%」として表示
- 「スコア」機能を廃止し、「確認履歴」へ変更
- 起床時刻と通知時刻を「予定」タブに集約
- `docs/testing.md` を履歴仕様に更新
- プロダクト判断メモを追加

## 背景

実機確認で、通知をタップしてアプリへ戻った際にクラッシュするケースがありました。

また、当初のスコア機能について、以下の違和感がありました。

- 点数稼ぎができる
- 昼に寝る人や夜勤ユーザーに合わない
- 充電忘れ防止という目的から少しズレる

そのため、ユーザーを採点するのではなく、確認した事実を残す「履歴」へ変更しました。

## 変更内容

### 通知タップ処理

- 通知デリゲートから直接画面状態を更新しないようにしました。
- `NotificationOpenTracker` で通知タップ状態を一時保存し、アプリ側で安全に消費します。

### ホーム画面

- 下部の更新ボタンをやめ、上部ツールバーの更新アイコンと引っ張って更新に変更しました。
- 最終更新時刻を表示します。
- バッテリー残量は「約xx%」として表示します。
- 最終確認履歴をホームに表示します。

### 履歴機能

- スコア/加点/7日平均を廃止しました。
- 履歴には以下を保存します。
  - 確認時刻
  - バッテリー残量
  - 充電状態
  - 確認のきっかけ
- 短時間の同一状態の自動更新は重複記録しないようにしました。

### ドキュメント

- READMEを履歴仕様に更新しました。
- `docs/testing.md` を最新QA観点に更新しました。
- `docs/product-review.md` を追加しました。

## 確認したこと

- `git diff --check` は通過。
- Windows環境のため、Xcodeビルドと実機検証は未実施。

## Mac/Xcodeで確認してほしいこと

- Xcodeでビルドできること。
- Unit Testが通ること。
- 通知タップでクラッシュしないこと。
- 通知タップ後に履歴へ「通知から確認」として記録されること。
- 手動更新で「手動更新」として履歴に残ること。
- アプリ復帰時の自動更新が短時間に増えすぎないこと。
- バッテリー状態表示が実機状態と大きく矛盾しないこと。
